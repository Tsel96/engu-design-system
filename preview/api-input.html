Input

A pill-shaped input with floating label, optional prefix and trailing slots, and helper / error text. Used for text, email, password, money, search and select fields — the variant is set by kind.

API

PropTypeDefaultDescription
labelrequiredstringundefinedFloating label above the value. Always required for accessibility.
valuestringundefinedControlled value. Pair with onChange.
defaultValuestringundefinedUncontrolled initial value.
onChange(v: string) => voidundefinedFires on every keystroke. Receives the new value, not the event.
placeholderstringundefinedShown when empty and not focused.
kind"text" | "email" | "password" | "money" | "search" | "select" | "date""text"Behavior + default trailing slot. password adds the eye toggle, search the magnifier + clear, select opens a menu.
size"compact" | "default""default"compact = 48px (label inline). default = 72px (floating label). Use compact when ≥4 fields stack.
state"default" | "focused" | "error" | "disabled" | "readonly""default"Visual state. Driven by the browser; force only for docs.
helperTextstringundefinedPersistent description below the field.
errorTextstringundefinedWhen set, overrides helperText and applies the error state.
prefixReactNodeundefinedLeading slot — icon or short string. Hidden in compact.
suffixReactNodeundefinedTrailing slot. Defaults set by kind.
disabledbooleanfalsePointer-events off, surface dimmed.
requiredbooleanfalseAdds the required indicator and forwards to the input.
autoCompletestringundefinedForwarded to the underlying input.
idstringautoForwarded; label uses htmlFor.

Usage

<Input
  label="Email"
  kind="email"
  value={email}
  onChange={setEmail}
  helperText="We will only contact you about your child."
/>
